# Adversary Model v0.1

## 攻撃者に許す能力

攻撃者は確率的多項式時間で動作し、次を行える。

- Parent A と Parent B の双方を支配する
- 全 Consensus / 保存 / 配信ノードを支配する
- 通信を盗聴、遅延、複製、遮断、改変する
- 正規 Child と公開証明を観察する
- Parent の申告を任意に変更する
- 偽の履歴を配布する
- 本物の Child を隠す、削除する、配信しない
- 複数セッションを並行実行する
- 使用済みメッセージを再送する
- 規定時点より後に一部鍵を取得する

## Phase 0 の信頼根

最初の中心定理では次を公理として置く。

- 検証者は真正な Oracle 公開根を少なくとも一つ保持する
- 対象 issuance の生成時点では Oracle 発行秘密状態が侵害されていない
- 基礎プリミティブの安全性仮定が成立する
- 正規検証アルゴリズム自体は改変されていない

## 別問題として分離する攻撃

次は真正性の改竄ではなく、別の安全性軸として扱う。

- 可用性攻撃
- 検閲
- 完全削除
- 時刻源停止
- Oracle のサービス拒否
- 検証者への偽ソフトウェア配布

## 攻撃成功

攻撃者が元と異なる系譜を持つ Child を作り、
真正な Oracle root を用いる正規検証者に
元と同一の正規 Child として受理させた場合、OCLM 改竄成功とする。
