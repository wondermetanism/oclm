# OCLM Phase 0 — Foundation

Oracle–Child Lineage Model（神託系譜モデル）の理論正本 v0.1。

## 目的

OCLM の思想を、反証可能な定義・攻撃者モデル・安全性命題へ固定する。
この Phase 0 では暗号方式や実装を先に決めず、「何を真実とし、何を破壊成功とするか」を定める。

## 中心仮説

親の申告は通常処理の前提である。
前提が崩壊・競合した場合、Oracle が生成前に固定した相補関係と、
Child が保持する時空系譜が最終的な正規性判定根拠となる。

## Phase 0 完了条件

- Truth Semantics が一義的である
- Oracle / Parent / Child / Time–Space の責任が分離されている
- 攻撃者の能力と禁止される公理破壊が明記されている
- 最初の中心定理と勝利条件が定義されている
- 未解決点が隠されず列挙されている

## 次段階

Phase 1 で最小状態機械を定義し、Phase 2 で Tamarin により反例探索を開始する。
