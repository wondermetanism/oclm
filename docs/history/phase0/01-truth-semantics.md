# Truth Semantics v0.1

## 1. 前提真実

Parent A と Parent B の申告が一致し、双方の承認が有効である間、
その申告は通常処理の前提として利用できる。

前提真実は効率のための運用上の真実であり、最終真実ではない。

## 2. 前提崩壊

次のいずれかが成立した場合、前提崩壊とする。

- Parent 間で申告が競合する
- Parent が過去の承認を否認する
- 保存ノード間で履歴が競合する
- 同一識別子について複数の正規候補が提示される
- 規定された監査条件が発火する

## 3. 系譜真実

前提崩壊後の正規性は、次の関係によってのみ判定する。

- Oracle が生成前に固定した発行根
- 一組の相補的資格
- Parent A と Parent B の参加証明
- Child が保持する生成系譜
- 生成時間窓
- 因果空間
- 一回性および世代順序
- 適用された生成規則の版

## 4. 真実の定義

OCLM における最終真実とは、申告内容そのものではなく、
Oracle が事前固定した生成可能性と Child の時空系譜が
正規検証規則の下で一致することである。

## 5. 同一性

親・Oracle 発行・生成時空・生成規則・相互作用のいずれかが異なる場合、
生成物は元の Child と同一ではない。

改変されたデータは存在できるが、元の Child の正規性を継承できない。
