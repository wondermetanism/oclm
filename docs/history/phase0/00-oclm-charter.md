# OCLM Charter v0.1

## 名称

Oracle–Child Lineage Model  
略称: OCLM  
日本語名: 神託系譜モデル

## 根本思想

OCLM は「どの申告を信じるか」を多数決で決める理論ではない。

Oracle が生成前に一組の相補的資格と許容時空を固定し、
二つの Parent がその資格を用いて相互作用した結果として Child が生成される。
Child は Oracle・両 Parent・生成時空・生成規則の系譜を保持する。

Parent の申告は通常処理を進めるための前提である。
申告が崩壊・競合・否認された場合、申告やノード投票ではなく、
Oracle と Child の系譜関係を正規性の最終判定根拠とする。

## OCLM が目指す保証

データを書き換えられないことではない。
書き換えられた対象を、元と同一の正規存在として成立させられないことを保証する。

## 非目標

- 人間の内心や誠実性の判定
- 現実世界のあらゆる命題の万能判定
- 特定の暗号アルゴリズムへの永久依存
- Consensus 多数派を真実の根とすること
