## Description

Oracle–Child Lineage Model (OCLM) is a formal model for
determining whether a state, record, right, or lineage can be
accepted as a legitimate continuation of an original truth
relationship.

OCLM does not define truth as merely the latest record or the
majority-accepted history. It defines legitimacy through
consistent lineage, verified relationships, and formally
specified acceptance conditions.

OCLM Phase 1 is formally verified and complete.

Phase 1-A through Phase 1-C6B have been completed.

Final verification results:

- 13/13 final Tamarin lemmas VERIFIED
- 15/15 verified release packages
- final proof set VERIFIED
- release integrity VERIFIED
- independent Phase 1-C6B reproduction: 13/13 VERIFIED

Verification environment:

- Tamarin Prover 1.12.0
- Maude 3.5.1

OCLM does not claim that physical records, storage media, or
databases cannot be modified, deleted, or forged.

It establishes, within the formally specified model, that an
inconsistent substitution cannot be accepted and continued as
the same legitimate truth and lineage.

Blockchain asks:

Which history should be accepted?

OCLM asks:

What can validly exist as truth and lineage?

OCLM is publicly verifiable and source-available for
non-commercial research, independent verification, and
educational use.

Commercial implementation, deployment, integration,
enterprise use, certification, and commercial services
