# OCLM AI Command Lineage Profile — Claims Matrix

| Claim | Current status | Evidence boundary |
|---|---|---|
| OCLM provides published formal foundations for canonical lineage and non-substitutability | Supported by frozen OCLM Phase 1 and Phase 2 | Applies only under the published models, assumptions, and compromise conditions |
| OCLM concepts can be specialized into an AI command-lineage architecture | Defined as an architectural profile in Phase AI-A | A dedicated AI refinement and Tamarin theory are still required |
| Every accepted agent action must descend from an authenticated root command | Phase AI-B proof target | Not yet established in an AI-specific model |
| Delegation cannot amplify authority | Phase AI-B proof target | Requires explicit subset and scope predicates |
| A tool cannot execute without an exact valid Action Certificate | Phase AI-B proof target | Requires deterministic gateway enforcement and no bypass path |
| Revoked commands and delegations cannot authorize later execution | Phase AI-B proof target | Requires monotonic revocation and modeled event ordering |
| An execution receipt corresponds to the exact certified action | Phase AI-B proof target | Requires receipt binding and trusted observation assumptions |
| OCLM prevents an AI from proposing an unauthorized action | Not supported | The profile governs legitimacy and execution, not internal proposal generation |
| OCLM perfectly interprets natural-language intent | Not supported | Command semantics and policy authoring remain an external boundary |
| OCLM makes an AI aligned, ethical, or benevolent | Not supported | Model alignment is outside the command-lineage proof |
| OCLM prevents every zero-day or sandbox escape | Not supported | It can deny legitimacy and gateway access; implementation vulnerabilities remain external |
| Every AI-agent implementation using OCLM is secure | Not supported | Implementation refinement and bypass resistance must be separately established |
| Every possible agentic attack has been cataloged | Not supported | Phase AI-A defines an initial bounded catalog |
| OCLM-AI is universally unbreakable | Not supported | Exceeds the formal evidence |
